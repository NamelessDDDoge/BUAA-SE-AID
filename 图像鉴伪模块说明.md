# 图像鉴伪模块说明

本文档描述 `ai-forensics` 仓库中图像鉴伪板块的目录结构、执行链路和核心逻辑。

## 1. 目标

图像鉴伪模块负责把用户上传或抽取得到的图片送入本地推理链，生成：

- 任务级记录
- 单图检测结果
- 子方法结果
- 可视化产物
- 最终 PDF 报告

这次重构后的目标是：

- 删除远程 GPU、Redis、Celery 依赖
- 保留现有后端任务与结果数据模型
- 将图像鉴伪收敛为纯本地执行链路

## 3. 后端主链

### 3.1 提交入口

前端调用 `/api/detection/submit/`，入口函数是 `submit_detection2`。

它主要负责：

1. 校验权限和配额
2. 查询 `ImageUpload`
3. 创建 `DetectionTask`
4. 记录 `Log`
5. 调用 `execute_detection_task`

### 3.2 本地批处理执行

`core/local_detection.py` 是现在的核心收敛层。

`execute_detection_task` 做的事情：

1. 为每张图片创建或更新 `DetectionResult`
2. 按批次生成临时目录
3. 为每个批次生成：
   - `img.zip`
   - `data.json`
4. 调用 `_run_local_detection_batch`
5. 批次结束后清理临时目录

### 3.3 后端到 AI 服务的桥接

`core/call_figure_detection.py` 负责：

- 准备 AI 服务输入目录
- 调用 `local_infer.py`
- 读取标准输出中的序列化结果
- 解码为 Python 对象返回给后端

现在它不再负责：

- 远程服务器连接
- SSH / SCP 文件传输
- 重连逻辑
- 远程任务触发

### 3.4 结果解析与落库

`_extract_single_result` 和 `_persist_detection_result` 负责把 AI 服务返回的原始结果映射到后端模型。

主要写入：

- `DetectionResult`
  - `is_fake`
  - `confidence_score`
  - `llm_judgment`
  - `ela_image`
  - `llm_image`
  - `exif_photoshop`
  - `exif_time_modified`
  - `status`
- `SubDetectionResult`
  - `method`
  - `probability`
  - `mask_image`
  - `mask_matrix`

同时会更新：

- `ImageUpload.isDetect`
- `ImageUpload.isFake`

### 3.5 任务完成

`_finalize_detection_task` 在批次全部完成后负责：

- 检查任务内图片是否全部完成
- 更新 `DetectionTask.status = completed`
- 写入 `completion_time`
- 生成任务报告

## 4. AI 服务内部逻辑

### 4.1 local_infer.py

`local_infer.py` 是后端直接调用的入口。

它负责：

- 读取输入图片和参数
- 调用 pipeline
- 将结果序列化输出给后端

### 4.2 pipeline_single_image.py

这一层负责组织单图检测的执行顺序，协调多个检测方法的输出。

### 4.3 SingleImageMethod.py

这一层是算法实现的主要装配点，负责调用各个具体检测方法。

后端最终只关心这些方法是否产出：

- LLM 文本结果
- ELA 图
- EXIF 判定
- 子方法 mask 与概率

## 5. 数据流

完整数据流如下：

1. 前端提交图片 ID 到 `/api/detection/submit/`
2. 后端创建 `DetectionTask`
3. 后端为每张图创建 `DetectionResult`
4. 后端生成批次 `img.zip` 和 `data.json`
5. 后端调用 `local_infer.py`
6. AI 服务执行 pipeline
7. AI 服务把结果回传给后端
8. 后端解析原始结果
9. 后端保存 ELA 图、mask 图并写入结果表
10. 后端更新 `ImageUpload.isDetect` / `ImageUpload.isFake`
11. 后端更新任务状态
12. 后端生成任务报告

## 6. 重构前后差异

### 重构前

- 图像鉴伪执行依赖 Redis + Celery
- 存在远程 AI 服务 / 远程 GPU 桥接
- 批次提交、拉取、重连逻辑分散
- 代码路径较脏，维护成本高

### 重构后

- 后端只走本地同步执行
- 图像鉴伪不再依赖 Redis / Channels / 远程桥接
- `core/local_detection.py` 成为统一执行入口
- `core/call_figure_detection.py` 只保留本地推理桥接职责

## 7. 当前边界

这次重构只清理图像鉴伪链路。

仍然保留但未在本轮彻底重构的部分：

- 论文检测异步链路
- 历史文档与部分乱码注释
- 算法权重管理方式

如果后续继续整理，优先建议：

1. 清理剩余乱码注释和历史说明文档
2. 统一训练代码与推理代码的目录约定
3. 把 AI 服务依赖和权重要求整理成单独文档
