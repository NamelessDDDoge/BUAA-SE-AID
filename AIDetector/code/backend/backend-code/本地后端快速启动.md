# 本地后端快速启动

## 1. 工作目录

在 `ai-forensics/code/backend/backend-code` 目录执行。

## 2. 环境准备

```powershell
conda activate detect
pip install -r requirements-dev.txt
```

## 3. 环境变量

```powershell
Copy-Item .env.example .env
```

至少确认以下配置：

```env
DATABASE_MODE=local
LOCAL_DB_NAME=db.sqlite3
AI_SERVICE_DIR=C:\Users\admin\Desktop\SE\BUAA-SE-AID\ai-forensics\code\ai-service\ai-service-code
AI_TEST_DIR=C:\Users\admin\Desktop\SE\BUAA-SE-AID\ai-forensics\code\ai-service\ai-service-code\test
```

如果要切换到已经配置好的阿里云 PostgreSQL，在 `.env` 中改成：

```env
DATABASE_MODE=aliyun_postgres
ALIYUN_DB_USE_SSH_TUNNEL=True
ALIYUN_DB_HOST=...
ALIYUN_DB_PORT=5432
ALIYUN_DB_NAME=...
ALIYUN_DB_USER=...
ALIYUN_DB_PASSWORD=...
ALIYUN_SSH_HOST=...
ALIYUN_SSH_PORT=22
ALIYUN_SSH_USER=...
ALIYUN_SSH_PASSWORD=...
```

说明：

- 默认仍然是本地 SQLite，不影响现有本地通路
- `ALIYUN_DB_USE_SSH_TUNNEL=False` 时，后端直接连接 RDS 地址
- `ALIYUN_DB_USE_SSH_TUNNEL=True` 时，后端启动时会自动建立 SSH 隧道
- 云端数据库结构以 Django migration 为准，目前来源于 `core/migrations/0001` 到 `0005`
- 图像鉴伪已经改为本地同步执行
- 不再需要 Redis
- 不再需要图像检测 Celery worker

## 4. 快速切换数据库

切到本地：

```powershell
powershell -ExecutionPolicy Bypass -File .\switch_database.ps1 -Mode local
```

切到阿里云：

```powershell
powershell -ExecutionPolicy Bypass -File .\switch_database.ps1 -Mode aliyun_postgres
```

## 5. 初始化数据库

```powershell
python manage.py migrate
python manage.py createsuperuser
```

如果要初始化阿里云数据库，优先直接运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\init_aliyun_database.ps1
```

这条命令会：

- 把 `.env` 中的 `DATABASE_MODE` 切到 `aliyun_postgres`
- 对阿里云数据库执行 `python manage.py check`
- 对阿里云数据库执行 `python manage.py migrate`

## 6. 启动后端

```powershell
python manage.py runserver
```

## 7. AI 服务要求

后端会直接调用 `local_infer.py`，因此需要保证：

- `AI_SERVICE_DIR` 指向 `ai-forensics/code/ai-service/ai-service-code`
- AI 服务目录依赖已经安装
- 模型权重已经放到算法要求的位置
